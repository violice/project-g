# Project Architecture - Feature-Sliced Design (FSD)

## Overview

This project follows the **Feature-Sliced Design (FSD)** architectural methodology for frontend applications.

## Layers Structure

```
src/
├── core/           # App layer - routing, entry points, global styles
├── routes/         # Pages layer - route components (TanStack Router file routes)
├── widgets/        # Widgets layer - autonomous UI blocks
├── features/       # Features layer - business features with UI and model
└── shared/         # Shared layer - reusable utilities, libraries, hooks
```

## Layer Descriptions

### `core/` (App Layer)

Entry point and global configuration.

- `main.tsx` - React app bootstrap
- `router.tsx` - TanStack Router configuration
- `route-tree.gen.ts` - Auto-generated route tree
- `styles.css` - Global styles (Tailwind + custom)

### `routes/` (Pages Layer)

TanStack Router file-based routes. Each file is a page component.

- `__root.tsx` - Root layout with devtools
- `index.tsx` - Home page (app entry point)
- `learn.tsx` - Learning page
- `tuner.tsx` - Tuner page

### `widgets/` (Widgets Layer)

Self-contained UI components that implement complete user scenarios.

- `tuner/` - Guitar tuner widget
  - `ui/` - UI components
  - `index.ts` - Widget public API

### `features/` (Features Layer)

Business features with user-facing value. Each feature has its own UI and model.

- `learn/` - Learning feature
  - `ui/` - Feature UI components (player, song-selector, visualizer, learn-page)
  - `model/` - Feature state management (TanStack Store)
  - `index.ts` - Feature public API

### `shared/` (Shared Layer)

Reusable code separated from business logic.

- `lib/` - Reusable libraries
  - `audio-pitch/` - Pitch detection (Karplus-Strong algorithm)
  - `music-player/` - Music playback engine
  - `tab-renderer/` - Tablature visualization
- `hooks/` - Custom React hooks
- `data/` - Static data (songs)
- `utils/` - Utility functions

## Import Rules

Imports must follow FSD hierarchy:

```
core/ → routes/ → widgets/ → features/ → shared/
```

- Pages (`routes/`) can import from `widgets/`, `features/`, and `shared/`
- Widgets (`widgets/`) can import from `features/` and `shared/`
- Features (`features/`) can import from `shared/`
- Modules cannot import from layers on the same or higher level

## Key Technologies

- **React 19** - UI framework
- **TanStack Router** - File-based routing
- **TanStack Store** - State management
- **Tailwind CSS v4** - Styling
- **pitchy** - Pitch detection library

## Notes

- `route-tree.gen.ts` is auto-generated by TanStack Router - do not edit manually
- Widgets represent complete, reusable UI blocks
- Features combine UI with business logic and state management
- Shared layer contains pure utility code without business context
- Features use flat structure in `ui/` folder for simple components
